TextField that allows for authentication of user input based on the manner in which a person types, including the way the phone is held and the speed at which the text is typed. Can be implemented within an application in order to allow a more secure login method.

Lock screen used to lock a device. The lock screen takes in a pattern drawn on a grid and compares it to a previously stored pattern. The stored pattern can contain multiple gestures drawn, and must be drawn in the same order to validate the password.

Access Control to add motion based events to an application. a motion threshold is set, and an event will trigger when that threshold is passed. This could allow an application to lock down when a sudden motion is detected, for example if someone grabs your device from you.

Access Control to add location based events to a target area. A center location is set, and different zones of a set radius can be created around that point. Continually updates location from the GPS of the device to accurately track location in regards to the zones. Uses the most accurate location information available at all times.

Lock screen used to lock a device. The lock screen is similar to the Android pin code lock screen, with 9 pins on the screen that the user draws a pattern between. The pattern can be as long as the user wants it to be, and can reuse pins that have been used in the password already.

Library that can be used to implement a lock screen. An image will be displayed that the user can tap a series of points. Those points can then be stored in the application as a password, and can later be compared to another set of touches to authenticate the user.

Lockscreen that captures 8 cardinal recognized gestures in sequence to store as a password. Shows path rubber banding for user feedback. Keeps track of all attempts to unlock, and has the capabilitiy to lock the application down after a particular amount of failed attempts. The passcode is stored by entering the same set of gestures twice in a row successfully.

Access Control to add camera events to an application. Camera Detection monitors the front facing camera in order to detect two different events: camera flashes and facial detection. A light change threshold can be set, so if the camera detects a flash of light, the application can be locked down. The camera also keeps track of how many faces are detected in the image, so that the application could be locked down if multiple people are looking at the phone, or no one is looking at the phone.
